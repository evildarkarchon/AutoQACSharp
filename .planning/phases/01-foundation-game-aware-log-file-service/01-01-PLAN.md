<objective>
Build the game-aware log file service contract and implementation that resolves correct xEdit log filenames from GameType and reads only new content via byte offset tracking.

Purpose: This is the foundation for the entire xEdit log parsing fix. Currently the service derives log filenames from the executable stem (uppercase) and uses timestamp-based staleness detection -- both are incorrect. This plan replaces the naming with xEdit's verified `{wbAppName}Edit_log.txt` convention and introduces offset-based reading that precisely isolates the current session's log output.

Output: New `LogReadResult` model, rewritten `IXEditLogFileService` interface with GameType-based API, and fully implemented `XEditLogFileService` with game-aware naming, offset capture, offset-based reading with truncation handling, and exponential backoff retry.
</objective>

**5. ReadLogContentAsync(string xEditDirectory, GameType gameType, long mainLogOffset, long exceptionLogOffset, CancellationToken ct):**
- Get main log path via `GetLogFilePath(xEditDirectory, gameType)`
- Get exception log path via `GetExceptionLogFilePath(xEditDirectory, gameType)`
- Read main log content via `ReadFromOffsetWithRetryAsync(mainLogPath, mainLogOffset, ct)` -- returns `string`
- Read exception log content via `ReadFromOffsetWithRetryAsync(exceptionLogPath, exceptionLogOffset, ct)` -- returns `string`
- Split main log content into lines: `content.Split('\n').Select(l => l.TrimEnd('\r')).Where(l => !string.IsNullOrEmpty(l)).ToList()` (handles both `\r\n` and `\n`)
- Build and return `LogReadResult` with:
  - `LogLines` = the parsed lines list
  - `ExceptionContent` = exception content if non-empty, else null
  - `Warning` = null (warnings come from callers interpreting empty results)
- If main log file does not exist after xEdit exit: return `LogReadResult { LogLines = [], ExceptionContent = null, Warning = "Main log file not found: {path}" }`

**6. Private helper `ReadFromOffsetWithRetryAsync(string filePath, long offset, CancellationToken ct)`:**
- If `!File.Exists(filePath)` return `string.Empty` (handles missing exception log gracefully)
- Retry loop: `maxRetries = 3`, `baseDelayMs = 100`
- For each attempt (0 to maxRetries):
  - Try: open `new FileStream(filePath, FileMode.Open, FileAccess.Read, FileShare.ReadWrite)`
  - If `offset > fs.Length`: reset offset to 0 (truncation handling per Pitfall 2 in research), log warning
  - `fs.Seek(offset, SeekOrigin.Begin)`
  - `using var reader = new StreamReader(fs)`
  - `return await reader.ReadToEndAsync(ct).ConfigureAwait(false)` (OFF-02)
  - Catch `IOException` when `attempt < maxRetries`: log debug, `await Task.Delay(baseDelayMs * (1 << attempt), ct).ConfigureAwait(false)` (OFF-04: 100ms, 200ms, 400ms)
  - Catch `IOException` on final attempt: log warning, return `string.Empty`

**7. Keep old obsolete methods delegating to new ones:**
```csharp
[Obsolete("Use GetLogFilePath(string, GameType) instead.")]
public string GetLogFilePath(string xEditExecutablePath)
{
    var dir = Path.GetDirectoryName(xEditExecutablePath);
    if (string.IsNullOrEmpty(dir))
        throw new ArgumentException("Invalid xEdit executable path.", nameof(xEditExecutablePath));
    var stem = Path.GetFileNameWithoutExtension(xEditExecutablePath).ToUpperInvariant();
    return Path.Combine(dir, $"{stem}_log.txt");
}

[Obsolete("Use ReadLogContentAsync instead.")]
public async Task<(List<string> lines, string? error)> ReadLogFileAsync(
    string xEditExecutablePath, DateTime processStartTime, CancellationToken ct = default)
{
    // Legacy implementation preserved for CleaningOrchestrator compatibility (Phase 3 replaces this call)
    var logPath = GetLogFilePath(xEditExecutablePath);
    if (!File.Exists(logPath))
        return (new List<string>(), $"Log file not found: {logPath}");
    var logModifiedUtc = File.GetLastWriteTimeUtc(logPath);
    if (logModifiedUtc < processStartTime.ToUniversalTime())
        return (new List<string>(), "Log file is stale (predates this cleaning run)");
    try
    {
        var lines = await File.ReadAllLinesAsync(logPath, ct).ConfigureAwait(false);
        return (lines.ToList(), null);
    }
    catch (IOException ex)
    {
        await Task.Delay(200, ct).ConfigureAwait(false);
        try
        {
            var lines = await File.ReadAllLinesAsync(logPath, ct).ConfigureAwait(false);
            return (lines.ToList(), null);
        }
        catch (IOException secondEx)
        {
            return (new List<string>(), $"Failed to read log file: {secondEx.Message}");
        }
    }
}
```

All async methods use `ConfigureAwait(false)`. All accept `CancellationToken ct = default` as last parameter. Use structured Serilog logging with `[LogFile]` tag prefix.
