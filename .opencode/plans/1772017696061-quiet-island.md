## Implementation Notes
- For active-process stop/force-stop tests, simulate `onProcessStarted` callback from mocked `ICleaningService.CleanPluginAsync` and pass a safe process handle (e.g., `Process.GetCurrentProcess()`), while mocking `IProcessExecutionService.TerminateProcessAsync` so no real kill occurs.
- Use `TaskCompletionSource` gates so stop actions execute only after `_currentProcess` has been assigned by orchestrator callback.
- For timeout retry tests, return timed-out `CleaningResult` for first attempt(s) and assert callback invocation count + terminal result semantics.
- For hang forwarding test, return a controllable `Subject<bool>` from `IHangDetectionService.MonitorProcess(...)` and assert `orchestrator.HangDetected` includes `true` during processing and `false` at cleanup.
- For config watcher tests, keep unique temp dirs and wait on explicit conditions/counters (avoid relying solely on fixed sleeps).
- For parser tests, remove permissive/no-op assertions and assert exact expected counts/behavior contract.
