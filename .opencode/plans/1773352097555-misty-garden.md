# Incremental approximation updates

## Goal

Show each plugin row's `Approx. ITM | UDR | Nav` preview in the main window as soon as that plugin's approximation finishes, instead of waiting for the full approximation batch to complete.

## Recommended approach

1. Keep approximation analysis sequential inside `PluginIssueApproximationService`; only change when results are surfaced.
2. Extend `IPluginIssueApproximationService.GetApproximationsAsync(...)` with an optional per-plugin callback so `ConfigurationViewModel` can receive results one at a time during the existing background refresh.
3. Add a single-plugin merge method in state service so one completed approximation updates only its matching row and leaves other rows in `Pending` state.
4. Keep the existing batch merge for final reconciliation and failure fallback, but use it only when the refresh finishes or the whole refresh fails.
5. Update `PluginListViewModel.OnStateChanged(...)` to patch changed rows instead of clearing/rebuilding the whole collection on every incremental update, so the UI does not flicker or lose selection.

## File changes

- `AutoQAC/Services/Plugin/IPluginIssueApproximationService.cs`
  - Add an optional per-plugin callback parameter to `GetApproximationsAsync(...)`.
- `AutoQAC/Services/Plugin/PluginIssueApproximationService.cs`
  - Invoke the callback after each plugin result is produced.
  - Preserve current per-plugin exception handling so failed analysis still yields `Unavailable` immediately.
  - Keep returning the full result list for final reconciliation.
- `AutoQAC/ViewModels/MainWindow/ConfigurationViewModel.cs`
  - Pass the callback into `GetApproximationsAsync(...)` from `RunApproximationRefreshAsync(...)`.
  - For each callback result, ignore stale/canceled refresh generations and merge only that plugin's approximation into state.
  - Keep the existing whole-refresh fallback to mark all pending approximations unavailable on fatal refresh failure.
  - Update the nested `NoOpPluginIssueApproximationService` signature to match.
- `AutoQAC/Services/State/IStateService.cs`
  - Add `MergePluginApproximation(PluginIssueApproximationResult approximation)`.
- `AutoQAC/Services/State/StateService.cs`
  - Implement single-plugin merge using the same matching rules as batch merge (`FullPath`, then `FileName`).
  - Preserve `IsSelected` and `IsInSkipList` when replacing plugin rows.
  - Leave `MergePluginApproximations(...)` in place for end-of-refresh reconciliation and full fallback behavior.
- `AutoQAC/ViewModels/MainWindow/PluginListViewModel.cs`
  - Replace collection clear/rebuild logic with indexed updates/removals/additions so incremental approximation updates are visible immediately without resetting unaffected rows.

## Behavior and edge cases

- Ignore late callback results from an older refresh generation.
- Ignore callback results after `CancelPendingApproximation()` cancels the current refresh.
- Preserve any already-computed rows when a refresh is canceled; do not let a stale refresh overwrite a newer plugin list.
- If the approximation service returns no results or throws before any per-plugin updates are delivered, keep the current fallback of marking all target rows unavailable.
- Do not change cleaning orchestration or plugin cleaning concurrency; this is preview-only behavior.
