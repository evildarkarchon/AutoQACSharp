   - Register the service in `AutoQAC/Infrastructure/ServiceCollectionExtensions.cs`.
   - Have the service accept `GameType`, resolved data folder, and cancellation token; internally map supported games to Mutagen `GameRelease`, build one shared load-order/link-cache context per refresh, analyze each plugin with `QueryPlugins.IPluginQueryService`, and return immutable per-plugin approximation snapshots keyed by stable plugin identity.

   - Merge against `CurrentState.PluginsToClean`, matching primarily on `FullPath` with case-insensitive comparison, so current `IsSelected` and `IsInSkipList` values survive background updates.
   - Keep failures non-fatal by merging `Unavailable` when approximation work is canceled or fails after the initial plugin list is already loaded.
4. Convert plugin refresh into a two-phase flow in `ConfigurationViewModel`.
   - Inject the new approximation service into `AutoQAC/ViewModels/MainWindow/ConfigurationViewModel.cs` and update `AutoQAC/ViewModels/MainWindowViewModel.cs` tests/constructor wiring as needed.
   - On Mutagen-backed refresh success, apply skip-list state, publish plugins immediately with approximation state set to `Pending`, then start background approximation work.
   - Track a refresh-scoped `CancellationTokenSource` plus a monotonic refresh/version marker so a newer refresh cancels older work and stale results cannot overwrite the latest list.
   - For file-based games, publish plugins with `Unavailable` approximation metadata and skip analysis entirely.
5. Surface the new metadata in the plugin list UI with minimal MVVM churn.
   - Update `AutoQAC/ViewModels/MainWindow/PluginListViewModel.cs` only as needed for binding convenience.
   - Update `AutoQAC/Views/MainWindow.axaml` so each row still shows the checkbox and filename, plus a compact secondary preview line for approximate `ITM`, `UDR`, and `Nav` counts when available.
   - Use explicit non-zero-looking text for non-available states; recommended default is `Analyzing...` for pending and `Preview unavailable` for unavailable.
6. Finish with focused tests and docs.
